## 기여 방법

### 🐛 버그 제보
- GitHub Issues에서 버그를 보고해 주세요
- 재현 단계, 예상 동작, 실제 동작을 포함해 주세요
- 스크린샷이나 화면 녹화가 있으면 더 좋습니다

### 💡 기능 제안
- GitHub Issues에서 새로운 기능을 제안해 주세요
- 어떤 문제를 해결하는지 설명해 주세요
- 가능하다면 구현 방법도 제안해 주세요

### 🎮 새 게임 추가
- 새로운 추첨 게임 아이디어가 있으신가요?
- 아래 [게임 개발 규격](#게임-개발-규격)을 따라 개발해 주세요

### 📝 문서 개선
- 오타 수정, 설명 개선 등 문서 기여도 환영합니다

## 개발 환경 설정

```bash
# 저장소 Fork 후 클론
git clone https://github.com/YOUR_USERNAME/lottery.git
cd lottery

# 로컬 서버 실행
python -m http.server 8000
# 또는
npx serve

# 브라우저에서 확인
open http://localhost:8000
```

## 게임 개발 규격

### 필수 요구사항

모든 게임은 다음 규격을 준수해야 합니다:

#### 1. 파일 구조
```
game-name/
└── index.html    # 단일 HTML 파일 (CSS/JS 임베디드)
```

#### 2. 메타데이터 주석

`index.html` 파일 최상단에 다음 형식의 주석을 필수로 포함:

```html
<!DOCTYPE html>
<!--
@game-meta
{
  "id": "game-folder-name",
  "title": "게임 제목",
  "emoji": "🎮",
  "category": "카테고리",
  "description": "게임 설명 (40-80자)",
  "features": ["🎯 특징1", "⚡ 특징2"],
  "color": "accent-color-name",
  "badge": "NEW|인기|null",
  "version": "1.0.0",
  "updated": "2025-01-01"
}
@end-game-meta
-->
<html lang="ko">
```

##### 메타데이터 필드 설명

| 필드 | 설명 | 예시 |
|------|------|------|
| `id` | 게임 폴더명 (URL 경로) | `"marble-olympics"` |
| `title` | 한글 게임 제목 | `"구슬 올림픽"` |
| `emoji` | 대표 이모지 1개 | `"🏅"` |
| `category` | 게임 분류 | `"레이싱"`, `"서바이벌"`, `"토너먼트"`, `"보드게임"`, `"회피게임"`, `"액션"` |
| `description` | 게임 설명 (40-80자) | `"구슬들이 다양한 트랙에서 경쟁하는..."` |
| `features` | 특징 배열 (2개 권장) | `["🗺️ 다양한 맵", "⚙️ 물리엔진"]` |
| `color` | 테마 색상 | `"marble"`, `"zombie"`, `"rps"`, `"bingo"`, `"chestnut"`, `"blue"`, `"green"`, `"purple"`, `"pink"`, `"orange"` |
| `badge` | 뱃지 (없으면 null) | `"NEW"`, `"인기"`, `null` |
| `version` | 시맨틱 버전 | `"1.0.0"` |
| `updated` | 최종 수정일 | `"2025-01-15"` |

#### 3. Sidebar 구성 규격

```
┌─────────────────────────┐
│ 🎮 게임 제목          ? │  ← 타이틀 + 도움말 버튼
├─────────────────────────┤
│ 👥 참여자 명단 (콤마 구분) │
│ ┌─────────────────────┐ │
│ │ 영수, 영호, 영식... │ │  ← textarea
│ └─────────────────────┘ │
├─────────────────────────┤
│ ⚙️ 게임 설정            │
│ 🏆 우승자 수    [3명 ▼] │  ← 설정 2-3개
│ ⚡ 게임 속도    [보통 ▼] │
├─────────────────────────┤
│ [  🎮 시작  ]    [ ↺ ] │  ← 시작/리셋 버튼
├─────────────────────────┤
│ 🏅 실시간 랭킹          │
│ 1. 🥇 영수      [생존]  │
│ 2. 🥈 영호      [생존]  │
│ 3. 🥉 영식      [탈락]  │
├─────────────────────────┤
│ [📋] [GitHub] [☕]     │  ← Footer 버튼
└─────────────────────────┘
```

#### 4. 기술 요구사항

- ✅ Single HTML 파일 (CSS/JS 임베디드)
- ✅ 모바일 반응형 디자인
- ✅ 다크 테마 (glassmorphism 효과)
- ✅ HTML5 Canvas 게임 렌더링
- ✅ localStorage 설정 저장
- ✅ 10-60초 내 게임 완료
- ✅ 실시간 랭킹 업데이트

새 게임 PR 전 확인:

- [ ] Copyright 주석 포함
- [ ] Sidebar 규격 준수

- [ ] Footer 버튼 포함

## Pull Request 가이드

### PR 제출 전

1. 최신 `main` 브랜치에서 시작
2. 의미 있는 커밋 메시지 작성
3. 테스트 완료 확인

### PR 제목 형식

```
[타입] 간단한 설명

예시:
[새게임] 경마 레이싱 게임 추가
[버그수정] 빙고 게임 모바일 레이아웃 수정
[개선] 포털 페이지 애니메이션 최적화
[문서] README 업데이트
```

### PR 본문 템플릿

```markdown
## 변경 사항
- 

## 스크린샷 (해당 시)


## 테스트
- [ ] 데스크톱 Chrome
- [ ] 데스크톱 Firefox
- [ ] 모바일 Safari
- [ ] 모바일 Chrome

## 체크리스트
- [ ] 코드 스타일 가이드 준수
- [ ] 게임 규격 준수 (새 게임인 경우)
- [ ] 문서 업데이트 (필요 시)
```
